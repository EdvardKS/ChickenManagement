Nombre del cliente

Botón de acción (🔘) que abrirá un Drawer con más información y botones de acción
🎯 Detalles importantes del botón de acción
El botón de acción abrirá un Drawer en el que se mostrarán todos los detalles del pedido y los botones de acción correspondientes.
Si el pedido requiere confirmación por WhatsApp, el botón debe mostrar una señal de notificación visual (como un ícono rojo con una alerta 🔴 o una burbuja de notificación).
Si el pedido tiene un número de teléfono, dentro del Drawer debe aparecer un botón que genere un enlace dinámico a WhatsApp Web o la app de WhatsApp en móvil con un mensaje personalizado.

El número de teléfono debe llevar el prefijo del país (34 para España o el que corresponda).
El texto debe codificarse en URL encoding (%20 para espacios, %0A para saltos de línea).
El mensaje puede personalizarse con el nombre del negocio.
✅ Acciones dentro del Drawer de cada pedido
Confirmar pedido (✔️) →
Resta la cantidad de pollos del pedido en "pollos actuales" en el stock.
El pedido desaparece de la tabla (ya que ha sido procesado).
Eliminar pedido (❌) →
El pedido se elimina de la tabla.
No se hacen modificaciones en el stock.
Marcar como error (⚠️) →
Se registra como error en la base de datos (para futuras estadísticas).
No se hace ningún ajuste en stock.
El pedido se elimina de la tabla.
📊 Actualización del Stock en tiempo real
Cuando se confirma un pedido, se debe restar la cantidad de pollos del stock en "pollos actuales".
Si se elimina o se marca como error, el stock no debe cambiar.
La tabla debe actualizarse automáticamente tras cada acción, asegurando que los cambios se reflejen sin necesidad de recargar la página.
🔗 Requerimientos Técnicos
Implementar en React con TypeScript.
Usar TanStack React Query para manejar las actualizaciones sin recargar la página.
Diseñar la tabla con Tailwind CSS o el sistema de estilos ya implementado.
Implementar el Drawer con shadcn/ui o el componente de Drawer ya existente.
Crear las rutas necesarias en la API para manejar las acciones:
PATCH /api/orders/:id/confirm → Para confirmar un pedido y actualizar el stock.
DELETE /api/orders/:id → Para eliminar un pedido sin afectar el stock.
PATCH /api/orders/:id/error → Para marcar un pedido como error y registrar la acción.
